## MoSCoW Analysis

### Must Have

- Global app state management
- API request state handling
- Mock API
- Clear presentation of parking zone data and violations
- Mobile-first layouts (`375px` to `428px` according to quick google search)

### Should Have

- Reusable component architecture
- Navigation into/out of data, e.g. overview + detail screens
- Screens for 4 key data types (zone, vehicle, alert, activity)

### Could Have

- Map View to aid with getting directions to parking zones
- Demo of the power of RTK Query's global request configuration, e.g. converting case of data when sending/receiving from APIs
- Way of converting Lat/Long location to address value or providing a link to a map app to open directions to Lat/Long

### Won't Have

- Live camera feed showing parking zones prior to PEO arriving onsite
